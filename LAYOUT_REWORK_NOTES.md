# Layout Rework Notes

Sprint 10A was a visual density reduction sprint, not a product-feature sprint.

## Home

- Reworked the first screen around one dominant Next Action block.
- Readiness is shown as quiet text rows with subtle icons instead of colored pills.
- Operational counts remain, but metric cards are neutral.
- Programme dots are neutral rather than programme-colored.
- Recent work uses simple text metadata instead of pills.

## Strategy

- Strategy is now the primary editable surface.
- Layout uses two-column editable panels with a persistent save bar.
- Programmes are editable inline with neutral controls.
- Risk/claims guidance is editable alongside programmes.

## Create

- Preserved real readiness bars because they represent actual completed production steps.
- Reduced decorative programme color in queue rows.
- Changed queue metadata to neutral text and stage labels.

## Calendar

- Weekly planner audit is visually secondary.
- Audit detail is collapsed by default.
- Large red/yellow/green numeric state treatment is reduced.
- Format/programme chips are neutralized.
- No progress bars were added.

## Pipeline

- Programme color rails and angle colors were neutralized.
- Next action remains visible.
- Warning/blocker state color remains available for real attention.

## Analyze

- Kept the Sprint 9D “Workspace signals” framing.
- No advanced analytics or predictive claims were added.
